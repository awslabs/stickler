    Development Environment Setup

    This guide walks you through setting up a complete development environment for contributing to Stickler.

    Prerequisites

    Required

    • Python 3.12+ - Stickler requires Python 3.12 or higher
    • Git - For version control

    Recommended

    • uv - For environment and dependency management
    • VS Code - IDE with Python support

    Verify Prerequisites

    # Check Python version
python --version  # Should be 3.12.x or higher

# Check Git
git --version

    Quick Setup

    Option A: uv (Recommended)

    uv handles Python installation, virtual environment creation, and dependency installation in one step.

    # Clone the repository
git clone https://github.com/awslabs/stickler.git
cd stickler

# Install everything (Python, venv, dependencies)
uv sync

    Option B: pip + venv

    Use Python's built-in virtual environment:

    # Clone the repository
git clone https://github.com/awslabs/stickler.git
cd stickler

# Create virtual environment (requires Python 3.12+ already installed)
python -m venv .venv

# Activate the environment
# On macOS/Linux:
source .venv/bin/activate
# On Windows:
.venv\Scripts\activate

# Install in development mode
pip install -e ".[dev]"

    Verify Installation

    After installation, verify everything is working:

    # Run tests
uv run pytest tests/ -v --tb=short
# or, if using pip + venv:
pytest tests/ -v --tb=short

# Check a quick example
python examples/scripts/quick_start.py

    You should see all tests passing.

    Development Dependencies

    The [dev] extras install these additional packages:

    Package Version Purpose
    pytest >=7.0.0 Testing framework
    pytest-xdist >=3.0.0 Parallel test execution
    coverage >=7.0.0 Code coverage reporting
    beautifulsoup4 >=4.14.2 HTML report testing

    Common Development Tasks

    Running Tests

    Tip: If you set up with pip + venv (Option B), drop the uv run prefix and run tools directly (e.g., pytest tests/).

    # Run all tests
uv run pytest tests/

# Run with verbose output
uv run pytest tests/ -v

# Run specific module
uv run pytest tests/structured_object_evaluator/

# Run specific test file
uv run pytest tests/structured_object_evaluator/test_comparators.py

# Run tests matching pattern
uv run pytest tests/ -k "levenshtein"

# Run with coverage
uv run coverage run -m pytest tests/
uv run coverage report -m

    Linting

    # Check for issues
uv run ruff check .

# Auto-fix issues
uv run ruff check --fix .

    Building Documentation

    The project uses MkDocs with Material theme for documentation.

    # Navigate to docs directory
cd docs

# Install documentation dependencies
make install

# Serve documentation locally with live reload
make docs
# Opens at http://127.0.0.1:8000

# Build static documentation
make build

# Clean generated files
make clean

    Documentation Dependencies (automatically installed via make install):

    • mkdocs - Static site generator
    • mkdocs-material - Material Design theme
    • mkdocs-awesome-nav - Automatic navigation from directory structure
    • mkdocstrings-python - Auto-generate API docs from docstrings
    • pymdown-extensions - Additional markdown extensions

    Available Makefile Targets:

    Target Command Purpose
    make install Install dependencies Install all documentation requirements
    make docs Serve with live reload Start local dev server at http://127.0.0.1:8000
    make build Build static site Generate production site in site/ directory
    make deploy Deploy to GitHub Pages Build and push to gh-pages branch
    make clean Remove build artifacts Clean the site/ directory

    Running Examples

    # Quick start example
python examples/scripts/quick_start.py

# JSON schema example
python examples/scripts/json_schema_demo.py

    Troubleshooting

    Common Issues

    Python Version Mismatch

    ERROR: Package requires Python >=3.12

    Solution: If using uv, it automatically installs the correct Python version: 

    uv sync

    If using pip, ensure you have Python 3.12+: 

    python --version

    Import Errors

    ModuleNotFoundError: No module named 'stickler'

    Solution: Install in development mode: 

    pip install -e ".[dev]"

    NumPy/SciPy Build Issues

    On older systems, you may encounter compilation errors:

    ERROR: Failed building wheel for numpy

    Solution: Pre-built wheels are available on PyPI. Ensure you're using a recent pip: 

    pip install --upgrade pip
pip install -e ".[dev]"

    See Known Issues for platform-specific problems (e.g., NumPy/GCC compatibility on RHEL).

    Permission Denied

    PermissionError: [Errno 13] Permission denied

    Solution: Don't use sudo. Use a virtual environment: 

    uv sync
# or
source .venv/bin/activate

    Tests Not Found

    no tests ran

    Solution: Run from project root: 

    cd /path/to/stickler
uv run pytest tests/

    Getting Help

    If you encounter issues not covered here:

    1. Check Known Issues
    2. Search existing GitHub Issues
    3. Open a new issue with:
    4. Your OS and Python version
    5. Full error message
    6. Steps to reproduce

    Next Steps

    Once your environment is set up:

    1. Read the Testing Guide to understand testing conventions
    2. Review the Code Style Guide for coding standards
    3. Check the Pull Request Guide when ready to contribute
